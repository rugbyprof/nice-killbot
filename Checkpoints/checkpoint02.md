# Checkpoint 2 — Autonomous Sensors

## Objective

Begin transitioning your robot from remote-controlled operation toward autonomous behavior.

Each team will receive one or more sensors to integrate into their robot.

---

## Available Sensors

Depending on availability, your robot may receive:

- Ultrasonic distance sensor (obstacle detection)
- Time-of-Flight (ToF) distance sensor
- Bumper switches
- GPS module

---

## Requirements

Integrate your assigned sensor(s) into your existing Raspberry Pi robot.

Your software should:

- Read sensor data
- Display or log useful values
- Demonstrate that the sensor is functioning correctly

If applicable, the sensor should influence robot behavior.

Examples include:

- Stop before hitting an obstacle.
- Detect collisions using bumper switches.
- Display current GPS coordinates.
- Report measured distances.

---

## Demonstration Checklist

Prepare to demonstrate:

- ☐ Sensor is wired correctly
- ☐ Sensor readings are valid
- ☐ Raspberry Pi software processes the readings
- ☐ Sensor data influences robot behavior (when appropriate)

---

## Looking Ahead

Checkpoint 2 serves as the bridge between a manually controlled robot and an autonomous one.

These sensors will become the foundation for our next phase of the course:

- Sensor fusion
- Autonomous navigation
- Decision making
- Introductory robotics AI
- Machine learning concepts

We will begin those topics in **Week 4**, giving everyone approximately **two weeks** to become comfortable with the new hardware before we start making robots that can think a little instead of just obeying commands.

Good luck—and remember: every robot has a personality. Some are just... more stubborn than others. 🤖
